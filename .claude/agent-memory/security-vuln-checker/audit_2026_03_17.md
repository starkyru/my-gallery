Security audit completed 2026-03-17 for new features (gallery-config, categories, bulk actions, archive, artist isActive).

**High**: Missing AdminGuard on GET /images/admin and POST /images/bulk-action -- artists can see all images and mutate any. BulkActionDto.action not constrained to allowlist.

**Medium**: Gallery config PUT accepts arbitrary JSONB keys (no DTO, Record<string,unknown> body). Public GET /artists returns inactive artists. GET /images/:id bypasses archive/isActive filters. Category update mass assignment risk (defense in depth). PUT /images/sort/order body is unvalidated plain array (no DTO class).

**Low**: No duplicate slug check before category create (raw DB error). No MaxLength on category name/slug. Frontend admin route guard is client-side only.

**Info**: Category image count includes archived images. Public gallery-config returns entire settings blob.

**Positive**: SQL injection safe (parameterized). No XSS (React). ValidationPipe whitelist active. CORS/Helmet in place. Auth guards pattern is correct where applied.

**Why:** Audit of newly added features to catch auth and validation gaps before deployment.
**How to apply:** Verify these fixes are applied in future PRs. Pattern to watch: new endpoints missing AdminGuard, plain Record<string,unknown> bodies bypassing ValidationPipe whitelist.

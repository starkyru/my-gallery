Security audit completed 2026-03-16. Key findings:

**Critical**: Hardcoded JWT secret fallback ('dev-secret-change-me') in jwt.strategy.ts/auth.module.ts/auth.service.ts allows token forging. Default admin password 'admin' in auth.service.ts onModuleInit.

**High**: ServicesController (services.controller.ts) missing AdminGuard -- photographers can modify payment credentials. Orders GET/:id and GET/:id/downloads have no auth (IDOR). Payment capture endpoint unauthed, no paymentId verification. BTCPay/Prodigi webhooks lack signature verification. Password reset tokens reusable (no invalidation after use).

**Medium**: No file type validation on upload (CWE-434). SSRF via BTCPay URL credential. CryptoService silently uses zero key. Reset token HMAC uses non-constant-time comparison (line 196 auth.service.ts). Download URL reuses JWT_SECRET. No rate limiting on login. timingSafeEqual crashes on length mismatch. No password complexity. Provider param not validated against allowlist. synchronize:true risk.

**Positive**: SQL injection safe (parameterized queries). No XSS in React components. Bcrypt cost=12. AES-256-GCM correctly implemented. Credentials never leaked to frontend. Helmet enabled. ValidationPipe with whitelist.

**Why:** First comprehensive audit of the codebase to identify exploitable vulnerabilities.
**How to apply:** Reference when reviewing future PRs touching auth, payments, webhooks, or file uploads. Verify fixes are applied.
